## 一、产品定位

### 1.1 核心理念

**"不做重复造轮子，做资源的调度者"**

本平台定位为测试人员的提效工具，核心价值在于：

- **调度整合**：整合 Apifox、云效等现有工具，统一入口管理
- **智能分析**：借助 AI 能力，提供测试分析和优化建议
- **经验沉淀**：构建知识库，实现经验复用和传承

### 1.2 目标用户

- 测试工程师：日常测试执行、用例管理、缺陷记录
- 测试负责人：测试规划、质量分析、团队管理
- 项目经理：质量报告、进度跟踪

### 1.3 核心价值

| 价值点 | 说明 |
|--------|------|
| 提效 | 自动化执行、智能分析，减少重复劳动 |
| 可视化 | 接口依赖可视化、执行趋势可视化 |
| 可追溯 | 执行记录、缺陷记录、知识沉淀全程可追溯 |
| 可复用 | 知识库支持经验复用，降低学习成本 |

## 三、功能模块详述

### 3.1 项目管理

#### 功能概述

项目管理是系统的核心模块，用于管理测试项目的基本信息和 Apifox 关联。

#### 功能清单

| 功能 | 说明 | 状态 |
|------|------|------|
| 项目列表 | 展示所有项目，支持搜索、筛选 | ✅ 已实现 |
| 创建项目 | 录入项目名称、描述、Apifox 项目 ID | ✅ 已实现 |
| 编辑项目 | 修改项目基本信息 | ✅ 已实现 |
| 删除项目 | 删除项目（软删除） | ✅ 已实现 |
| 项目详情 | 展示项目统计、测试套件、执行历史 | ✅ 已实现 |
| Apifox 同步 | 同步 Apifox 项目下的测试套件和测试场景 | ✅ 已实现 |

#### 项目详情页

项目详情页包含以下信息区块：

1. **基本信息卡片**
   - 项目名称、描述
   - Apifox 项目 ID
   - 创建时间、更新时间

2. **统计概览卡片**
   - 测试套件数量
   - 测试场景数量
   - 执行次数
   - 通过率
   - 平均执行耗时

3. **执行趋势图表**
   - 最近 7 天执行趋势
   - 通过/失败数量对比

4. **测试套件/场景列表**
   - 从 Apifox 同步的测试项
   - 支持保存、执行操作
   - 查看执行历史

5. **执行历史列表**
   - 该项目下的执行记录
   - 状态、结果、时间

6. **项目设置**
   - 编辑项目信息
   - 配置 Apifox Token

```

       │
       ▼
┌──────────────┐

└──────────────┘
       │
       ▼
┌──────────────┐
│ 调用 Apifox  │ ← CLI 自动执行前置脚本、后置脚本
│ CLI 执行测试  │
└──────────────┘
       │
       ▼
┌──────────────┐
│ 解析测试报告  │ ← 提取统计信息
└──────────────┘
       │
       ▼
┌──────────────┐
│ 存储报告     │ ← MySQL (摘要) + MongoDB (详情)
└──────────────┘
       │
       ▼
┌──────────────┐
│ 更新执行状态  │
└──────────────┘
```

### 3.3 定时任务

#### 功能概述

定时任务模块支持按计划自动执行测试，实现无人值守的自动化回归测试。

#### 功能清单

| 功能 | 说明 | 状态 |
|------|------|------|
| 创建定时任务 | 配置任务名称、调度规则、测试项 | 📋 规划中 |
| 调度类型支持 | Cron 表达式、固定间隔、单次执行 | 📋 规划中 |
| 任务启用/禁用 | 控制任务是否执行 | 📋 规划中 |
| 手动触发 | 立即执行一次任务 | 📋 规划中 |
| 执行历史 | 查看任务的执行记录 | 📋 规划中 |
| 通知配置 | 配置执行结果通知 | 📋 规划中 |

#### 调度类型

| 类型 | 配置方式 | 示例 |
|------|----------|------|
| Cron 表达式 | 标准 Cron 语法 | `0 2 * * *` (每天凌晨 2 点) |
| 固定间隔 | 间隔分钟数 | 每 60 分钟 |
| 单次执行 | 指定时间 | 2026-02-20 10:00:00 |

#### 常用 Cron 表达式

| 表达式 | 说明 |
|--------|------|
| `0 2 * * *` | 每天凌晨 2 点 |
| `0 */2 * * *` | 每 2 小时 |
| `30 9 * * 1-5` | 工作日早上 9:30 |
| `0 0 * * 0` | 每周日午夜 |
| `*/15 9-18 * * *` | 工作时间每 15 分钟 |
| `0 8 * * 1` | 每周一早上 8 点 |

#### 任务执行流程

```
┌──────────────┐
│  调度器触发   │ ← APScheduler 按 Cron 触发
└──────────────┘
       │
       ▼
┌──────────────┐
│  检查任务状态 │ ← 是否启用、是否在执行中
└──────────────┘
       │
       ▼
┌──────────────┐
│  创建执行记录 │
└──────────────┘
       │
       ▼
┌──────────────┐
│  执行测试    │ ← 调用 Apifox CLI
└──────────────┘
       │
       ▼
┌──────────────┐
│  存储报告    │
└──────────────┘
       │
       ▼
┌──────────────┐
│  发送通知    │ ← 钉钉/企微/邮件
└──────────────┘
       │
       ▼
┌──────────────┐
│  更新任务状态 │ ← 更新 last_run_at, next_run_at
└──────────────┘
```

### 3.4 接口编排

#### 功能概述

接口编排模块提供可视化的接口依赖关系配置，解决接口测试中依赖关系难以梳理的痛点。

#### 功能清单

| 功能 | 说明 | 状态 |
|------|------|------|
| 从 Apifox 导入 | 拉取项目下所有接口作为节点 | 📋 规划中 |
| 可视化编排 | 拖拽节点、连线配置依赖 | 📋 规划中 |
| 数据映射 | 配置接口间的数据传递关系 | 📋 规划中 |
| 编排保存 | 保存编排配置 | 📋 规划中 |
| 生成测试场景 | 将编排导出为 Apifox 测试场景 | 📋 规划中 |
| 一键执行 | 按编排顺序执行接口 | 📋 规划中 |
| 依赖分析 | 自动分析接口依赖关系 | 📋 规划中 |

#### 痛点解决

| 传统方式 | 编排方式 |
|----------|----------|
| 阅读文档理解依赖 | 可视化查看依赖关系 |
| 容易遗漏依赖 | 连线明确依赖关系 |
| 测试顺序混乱 | 按编排顺序执行 |
| 数据传递不清晰 | 数据映射一目了然 |

#### 编排画布设计

```
┌─────────────────────────────────────────────────────────────────────────┐
│  接口编排画布                                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│    ┌──────────┐         ┌──────────┐         ┌──────────┐             │
│    │ 登录接口  │ ──────→ │ 获取用户  │ ──────→ │ 修改信息  │             │
│    │ POST/login│         │ GET/user │         │ PUT/user │             │
│    └──────────┘         └──────────┘         └──────────┘             │
│         │                    │                    │                    │
│         │                    ↓                    │                    │
│         │              ┌──────────┐              │                    │
│         │              │ 查询订单  │              │                    │
│         │              │ GET/order│              │                    │
│         │              └──────────┘              │                    │
│         │                    │                    │                    │
│         ▼                    ▼                    ▼                    │
│    [token]              [user_id]           [更新结果]                  │
│                                                                         │
├─────────────────────────────────────────────────────────────────────────┤
│  节点面板: [登录] [用户] [订单] [支付] [商品] ...                        │
│  操作: [从 Apifox 导入] [保存编排] [生成测试场景] [执行]                 │
└─────────────────────────────────────────────────────────────────────────┘
```

#### 数据映射示例

```
登录接口 (source) → 获取用户接口 (target)

数据映射:
  target.header.Authorization = "Bearer " + source.response.data.token
  target.path.user_id = source.response.data.user_id
```
